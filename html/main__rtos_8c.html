Detailed Description

Aplicación principal del Sistema de Control de Acceso con FreeRTOS y Display SSD1306.

Author
Sistema de Control de Acceso
Date
2025

Sistema de control de acceso completo que utiliza FreeRTOS para manejo concurrente de:

  • Teclado matricial 4x4
  • Sistema de LEDs de señalización
  • Display SSD1306 I2C para interfaz de usuario
  • Base de datos de usuarios
  • Control de acceso con máquina de estados

Hardware requerido:

  • Raspberry Pi Pico
  • Teclado matricial 4x4
  • 3 LEDs (Verde, Rojo, Amarillo)
  • Display SSD1306 OLED I2C 128x32
  • Resistencias limitadoras de corriente para LEDs

Conexiones:

  • Teclado: Filas GP6-GP9, Columnas GP10-GP13
  • LEDs: Verde GP15, Rojo GP16, Amarillo GP17
  • Display SSD1306: SDA GP4, SCL GP5

int main ( )

Función principal del sistema con FreeRTOS.

Inicializa todos los módulos del sistema, crea las tareas de FreeRTOS y inicia el scheduler del sistema operativo.

Returns
int Código de salida (nunca se alcanza en sistemas embebidos)

Inicialización de comunicación serie USB Timeout extendido para establecer conexión

Inicialización de módulos del sistema Orden optimizado para el sistema con FreeRTOS

Crear tareas de FreeRTOS

Cada módulo funcional del sistema se ejecuta como una tarea independiente, permitiendo procesamiento concurrente y comunicación através de colas.

Iniciar el scheduler de FreeRTOS

Una vez que se inicia el scheduler, el control pasa al kernel de FreeRTOS y las tareas comienzan a ejecutarse según sus prioridades y algoritmos de scheduling configurados.

Esta función nunca retorna en operación normal.

El código siguiente solo se ejecuta si hay un error crítico en el sistema que causa que el scheduler falle
